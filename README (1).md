# Cluj Mall AI Assistant — Semantic Web Project

A proxy-MCP application that lets a user ask natural-language questions about shopping malls in Cluj-Napoca (stores, locations, addresses, statistics) and get answers generated by an LLM (Gemini), based on data read/written across three different data sources: **JSON-Server**, **JSON-GraphQL-Server**, and **RDF4J** (a semantic triplestore, queried via SPARQL).

## Architecture

```
┌──────────┐        ┌───────────────────────────────┐
│          │  (0)   │             MCP Proxy          │
│ Frontend │───────►│  ┌───────────┐   JSON-RPC  ┌──────────┐
│  (AJAX)  │◄───────│  │MCP client │◄───────────►│MCP server│
│          │  (8)   │  │ (Gemini)  │              │          │
└──────────┘        │  └───────────┘              └────┬─────┘
                     └──────────────────────────────────┼──────┘
                                                          │
                     ┌────────────────────────────────────┼───────┐
                     ▼                    ▼                ▼
             JSON-Server          JSON-GraphQL-Server     RDF4J
             (localhost:4000)     (localhost:3000)   (localhost:8080)
```

**Flow:**
1. The user types a question in the frontend (`index.html`), which sends a `POST` request to the MCP client (`/ask`).
2. The MCP client asks the MCP server for the list of available tools (JSON-RPC, port `6000`), then asks Gemini which tool matches the user's request.
3. The MCP client executes the chosen tool through the MCP server, which in turn queries the right data source (JSON-Server / GraphQL / RDF4J via SPARQL).
4. The result is sent back to Gemini to produce a friendly natural-language answer.
5. The final answer is returned to the frontend.

The MCP proxy is split into an **MCP client** (handles communication with the LLM and the frontend) and an **MCP server** (handles communication with the three data sources), with the two communicating over **JSON-RPC**.

## Prerequisites

- **Node.js** (v18+)
- **json-server** and **json-graphql-server**
- **Apache Tomcat + RDF4J** (running locally)
- A **Gemini** API key (Google AI Studio)
- Node packages used in `proxy_mcp/`: `express`, `jayson`, `axios`, `cors`

## Setup & run (step by step)

### 1. Install dependencies
Inside `proxy_mcp/`:
```bash
npm install express jayson axios cors
```

### 2. Add your API key
Open `proxy_mcp/mcp_client.js` and set your own Gemini API key where indicated in the file.

### 3. Start JSON-Server (port 4000)
From the `data/` folder:
```bash
json-server --watch db.json --port 4000
```

### 4. Start JSON-GraphQL-Server (port 3000)
From the `data/` folder:
```bash
json-graphql-server db.js --port 3000
```

### 5. Start RDF4J (port 8080)
- Start Tomcat (`startup.bat` from the RDF4J bin folder).
- Create a new repository named **`grafexamen`**.
- Import the data from `data/turtle.txt` into that repository.

### 6. Start the MCP server
From `proxy_mcp/`:
```bash
node mcp_server.js
```
→ runs on `http://localhost:6000`

### 7. Start the MCP client
In a separate terminal, from `proxy_mcp/`:
```bash
node mcp_client.js
```
→ runs on `http://localhost:5000`

### 8. Start the frontend
From `frontend/`:
```bash
npx serve .
```
Open the shown address in **Chrome**.

## Available MCP tools

| Tool | Description | Data source |
|---|---|---|
| `get_mall_location_rdf` | address of a specific mall | RDF4J |
| `search_shops_by_mall_name_rdf` | stores inside a given mall | RDF4J |
| `find_shop_location_global` | where a specific store is located | RDF4J |
| `get_stores_by_floor_rdf` | stores filtered by floor/location | RDF4J |
| `get_mall_statistics_rdf` | number of stores per mall | RDF4J |
| `find_mall_by_address_rdf` | mall matching an address/street | RDF4J |
| `search_shops_by_keyword_rdf` | stores matching a keyword | RDF4J |
| `get_stores_by_mall_json` | stores in a mall (by id) | JSON-Server |
| `add_new_store_json` | add a new store | JSON-Server |
| `get_mall_hierarchy_gql` | full hierarchy of a mall | GraphQL |
| `add_store_graphql` | add a new store | GraphQL |
| `add_new_mall_rdf` | add a new mall (SPARQL INSERT) | RDF4J |
